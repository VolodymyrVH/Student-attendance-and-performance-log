# Student-attendance-and-performance-log
# Project work. A project to create a website for creating schedules, viewing attendance statistics, and student performance.

## ВСТУП

У сучасних умовах цифрової трансформації освіти дедалі більшої актуальності набуває автоматизація процесів управління навчальною діяльністю. Зростання кількості студентів, дисциплін, навчальних груп та викладачів створює потребу у створенні зручних програмних рішень, які дозволяють ефективно планувати навчальний процес, вести облік відвідуваності та контролювати рівень успішності студентів.

Традиційні методи формування розкладу занять, обліку відвідуваності та оцінювання студентів, що здійснюються у паперовому вигляді або за допомогою розрізнених електронних таблиць, є трудомісткими, схильними до помилок та не забезпечують оперативного доступу до аналітичної інформації. Це ускладнює управління навчальним процесом, створює додаткове навантаження на викладачів і адміністрацію, а також унеможливлює швидке реагування на зміни чи проблеми, які виникають у процесі навчання.

З метою усунення зазначених недоліків було розроблено застосунок для формування розкладу, перегляду статистики відвідуваності та успішності студентів. Даний програмний продукт призначений для автоматизації ключових процесів навчальної діяльності та забезпечення зручного інтерфейсу для роботи користувачів різних ролей — викладачів, адміністраторів і студентів.

Розробка такого застосунку має практичну цінність, оскільки дозволяє не лише оптимізувати процеси планування та контролю, але й сприяє підвищенню дисципліни серед студентів. Своєчасний доступ до інформації про пропуски та оцінки мотивує студентів до більш відповідального ставлення до навчання, а викладачам дає змогу краще відстежувати динаміку успішності та своєчасно виявляти проблемні моменти.

З технічного погляду, застосунок може бути реалізований із використанням сучасних технологій програмування та баз даних, що забезпечує стабільність роботи, масштабованість і зручність у користуванні. Залежно від поставлених вимог, програму було реалізовано як веб-застосунок, що дає можливість адаптувати її до конкретних потреб навчального закладу.

Розроблений застосунок може бути інтегрований у внутрішню інформаційну систему навчального закладу або використовуватися як окремий інструмент. Він дозволяє отримувати звіти у зручному форматі (таблиці, графіки), що полегшує аналітичну роботу адміністрації.

Таким чином, створення застосунку для формування розкладу, перегляду статистики відвідуваності та успішності студентів є актуальним завданням, яке відповідає сучасним потребам цифрової освіти.

 
## ПОСТАНОВКА ЗАДАЧІ

Метою розробки є створенням веб-застосунку, який буде дозволяти викладачам, студентам і адміністрації навчального закладу зручно вести, переглядати та аналізувати інформацію про відвідування занять та успішність студентів у реальному часі. Додаток має забезпечити зручну та швидку взаємодію з користувачем, а також автоматизувати процеси обліку успішності, відвідуваності та формування аналітичних рекомендацій на основі історії навчання студента. Завдання, які необхідно виконати для досягнення поставленої мети:

1)	розробка структури бази даних: проектування реляційної бази даних для зберігання інформації про успішності та відмітки студентів або учнів, розклади занять. Реалізація таблиць для зберігання необхідних даних і створення зав’язків між ними; 

2)	розробка архітектури серверної частини: створення серверної частини, що реалізує API для обробки запитів користувачів, зберігання та обробку даних та інтеграцію з базою даних;

3)	розробка користувацького інтерфейсу: створення інтуїтивно зрозумілого та зручного інтерфейсу, що дозволяє користувачу легко взаємодіяти з функціями платформи, такими як перегляд навчальних занять, списків студентів, списки оцінок та присутності.

###	Опис бази даних
База даних слугує основою для збереження інформації, необхідної для роботи платформи. Вона містить дані про користувачів, предмети, уроки, оцінки, групи та успішність студентів. Система побудована на реляційній моделі, де дані організовані у таблицях зі встановленням зв’язків між ними за допомогою первинних та зовнішніх ключів. 

Ключовим аспектом є те, що кожен об'єкт, наприклад уроки або користувачі, мають окремі таблиці. Це дозволяє уникати надмірного дублювання інформації, зберігаючи її цілісність. Завдяки зовнішнім ключам забезпечується взаємозв’язок між таблицями, що дає змогу виконувати складні запити для отримання необхідних даних, наприклад, відомостей про оцінки й відвідуваність студента, або розклад і деталі уроків.
